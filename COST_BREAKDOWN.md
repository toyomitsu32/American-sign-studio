## 🎯 コスト削減の具体策

### 1. 音声認識 = ¥0/月

**使用技術**: Web Speech API（ブラウザ標準機能）

```javascript
// ブラウザの無料機能を使用
const recognition = new webkitSpeechRecognition();
recognition.lang = 'ja-JP';
recognition.start(); // ← 完全無料！
```

**メリット**:
- Google Speech-to-Text API（有料）を使わない
- Chrome、Safari、Edgeで標準サポート
- リアルタイム認識
- 追加コスト ¥0

**制限事項**:
- Firefox は別途対応が必要
- オフラインでは動作しない

→ **95%以上のユーザーで無料で動作**

### 2. AIアドバイス = ¥2,000〜3,000/月

**戦略**: 「事前生成 + キャッシング」で大幅削減

#### 従来の方法（高コスト）
```
ユーザーがリクエスト
  ↓
毎回GPT-4 APIを呼び出し ← 1回¥15〜30
  ↓
月1,000回 = ¥15,000〜30,000 😱
```

#### 提案する方法（低コスト）
```
【事前準備】
テンプレート × 色 × サイズ の組み合わせ（約200パターン）
を事前にGPT-4で生成してデータベースに保存

【実際の使用時】
データベースから取得 ← ¥0
  ↓
新しいパターンのみAPIを呼ぶ ← 月100回程度
  ↓
月100回 = ¥1,500〜3,000 ✅
```

**具体例**:
```json
// 事前生成して保存
{
  "template": "駐車場標識",
  "background": "yellow",
  "advice": "黄色と黒の組み合わせは視認性が高く、
             駐車場標識として最適です。
             テキストは24pt以上を推奨します。"
}
```

**追加コスト削減**:
- キャッシュ期間: 30日
- 同じ質問には保存済み回答を返す
- ピーク時のAPI呼び出し制限

### 3. 3Dプレビュー = ¥0/月

**使用技術**: Three.js（オープンソース、完全無料）

**配信方法**:
```
3Dモデルファイル（.glb形式）
  ↓
CDN経由で配信（Cloudflare R2 無料枠）
  ↓
ブラウザ側でレンダリング
```

**コスト**:
- Three.jsライブラリ: 無料
- 3Dモデル作成: 初期費用のみ
- 配信: CDN無料枠内（月10GB）
- レンダリング: ユーザーのブラウザで実行

→ **完全に¥0/月で運用可能**

### 5. データベース = ¥0/月

**使用サービス**: Firebase Firestore 無料プラン

**無料枠**:
- ストレージ: 1GB
- 読み取り: 5万回/日
- 書き込み: 2万回/日

**必要な容量**:
```

事前生成AIアドバイス: 200件 × 5KB = 1MB
注文データ: 月200件 × 50KB = 10MB
合計: 約12MB/月 ← 1GBの1.2%のみ使用
```

→ **数年間は無料枠内で運用可能**
