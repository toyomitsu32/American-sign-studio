# プロジェクトサマリー（日本語版）

## 📌 プロジェクト概要

**プロジェクト名**: SignCraft AI  
**サービス概要**: AI音声入力を活用したアメリカ標識のカスタムデザイン・制作依頼プラットフォーム

### 🎯 解決する課題

現在の標識制作サービスの課題：
- テンプレート選択後のカスタマイズが複雑
- デザインの完成イメージが掴みにくい
- 制作会社とのやり取りに時間がかかる
- デザインスキルが必要

### 💡 提供するソリューション

SignCraft AIの特徴：
1. **簡単操作**: テンプレート選択 → 音声でカスタマイズ → 注文（5分）
2. **AIアシスト**: 音声入力を理解してデザインに自動反映
3. **リアルタイムプレビュー**: 3Dで実際の完成イメージを確認
4. **ワンストップ**: 制作会社への直接発注まで一気通貫

## 🌟 主要機能

### 1. テンプレート選択
- 6つのカテゴリーから選択
  - 安全・警告標識
  - 駐車場・交通標識
  - ビジネス・店舗標識
  - 案内・情報標識
  - プライベート・住宅用
  - カスタムデザイン

### 2. AI音声カスタマイズ
**使い方**:
```
ユーザー: 「タイトルをWelcomeに変更して、背景を青にして」
システム: → 即座にデザインを反映
```

**対応コマンド例**:
- テキスト変更: "タイトルを〇〇に変更"
- 色変更: "背景を青にして"
- サイズ変更: "もっと大きくして"
- スタイル変更: "太字にして"

### 3. リアルタイム3Dプレビュー
- 360度回転可能
- 実際の設置環境での合成表示
- サイズ感の視覚化
- 昼/夜の見え方確認

### 4. 自動見積もり・即時注文
- サイズ・素材・数量に応じた即時見積もり
- 製作期間の自動計算
- 配送オプション選択
- ワンクリック注文

## 🏗️ 技術スタック

### フロントエンド
- **フレームワーク**: React 18 + Next.js 14 + TypeScript
- **スタイリング**: Tailwind CSS
- **3D描画**: Three.js / React Three Fiber
- **状態管理**: Zustand
- **音声入力**: Web Speech API + Google Speech-to-Text API

### バックエンド
- **サーバー**: Node.js + Express
- **データベース**: PostgreSQL (Supabase)
- **キャッシュ**: Redis
- **ストレージ**: Cloudflare R2
- **AI**: OpenAI GPT-4 + Google Speech-to-Text

### インフラ
- **ホスティング**: Vercel (フロント), AWS ECS (バック)
- **CDN**: Cloudflare
- **決済**: Stripe
- **監視**: Datadog

## 💬 FAQ

### Q: なぜ音声入力が必要なのですか？

**A**: 標識のカスタマイズは、「ここの文字を変えて、色をこうして...」という直感的な指示が多いため、音声入力が最も自然で効率的です。また、差別化要因にもなります。

### Q: 3Dプレビューのコストは妥当ですか？

**A**: Three.jsは軽量で、適切に最適化すれば追加コストは最小限です。ユーザー体験の向上による変換率アップで十分にペイします。

### Q: AI APIのコストが予想を超えたら？

**A**: キャッシング戦略、使用量制限、段階的な課金モデルで対応可能です。また、コスト監視アラートを設定します。

### Q: 競合が同様の機能を追加したら？

**A**: 継続的な機能改善、ユーザーエクスペリエンスの最適化、制作会社との深い統合により、先行者優位を維持します。

### Q: MVP後の方向転換は可能ですか？

**A**: はい。6週間のMVPでユーザーフィードバックを得て、Phase 2以降の投資判断ができる設計になっています。
